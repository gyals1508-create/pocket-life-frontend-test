# 통계 페이지 장바구니 완료율 데이터 미표시 문제

## 문제 상황
통계 페이지의 장바구니 완료율 차트에서 구매완료된 물품 데이터가 표시되지 않습니다.

**예시 상황:**
- 2026년 01월 20일: 장바구니 페이지에서 구매완료된 물품 2개 있음
- 통계 페이지: 장바구니 완료율이 0%로 표시됨 (데이터 없음)

## 현재 프론트엔드 동작
1. `/api/stats/cart/range?startDate=...&endDate=...` API 호출 시도
2. 실패 시 fallback: 기간 내 각 날짜별로 `/api/stats/cart?date=YYYY-MM-DD` 호출
3. 각 날짜별 `totalQuantity`와 `purchasedQuantity`를 누적하여 계산

## 확인 요청 사항

### 1. `/api/stats/cart/range` API
- 해당 API가 정상적으로 작동하는지?
- 구매완료된 항목(`isBought: true`)을 제대로 카운트하여 `purchasedQuantity`에 포함하는지?
- 응답 형식이 `{ purchaseRate, totalQuantity, purchasedQuantity }`인지?

### 2. `/api/stats/cart?date=YYYY-MM-DD` API
- 날짜별로 호출할 때 해당 날짜의 장바구니 통계를 제대로 반환하는지?
- 구매완료된 항목을 `purchasedQuantity`에 포함하는지?
- 응답 형식이 `{ purchaseRate, totalQuantity, purchasedQuantity }`인지?

### 3. 데이터 일관성
- 장바구니 페이지(`/api/cart?date=...`)에서 `isBought: true`인 항목이 있는데
- 통계 API(`/api/stats/cart?date=...`)에서 `purchasedQuantity`가 0인 경우가 있는지?
- 이 경우 원인은 무엇인지?

## 예상 원인 (추정)
1. 통계 API가 `isBought` 필드를 제대로 확인하지 않음
2. 날짜 필터링이 제대로 작동하지 않음
3. API 응답 구조가 프론트엔드가 기대하는 것과 다름

## 참고
프론트엔드에서는 디버깅 로그를 추가하여 각 날짜별 장바구니 통계 데이터를 콘솔에 출력하도록 수정했습니다.
백엔드 로그에서도 해당 날짜(2026-01-20)의 장바구니 통계 조회 요청이 들어오는지, 그리고 어떤 응답을 보내는지 확인 부탁드립니다.
